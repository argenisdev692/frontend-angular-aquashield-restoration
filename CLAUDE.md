# Frontend Project - Angular 22

## Architecture
- **Framework:** Angular 22 with Standalone Components (default) + **zoneless** change detection
- **TypeScript:** TypeScript 6 required — strict mode + `strictTemplates` (both default in v22)
- **State Management:** Signals (signal, computed, effect) + resource/rxResource/httpResource (all stable in v22) for async
- **Styling:** PrimeNG v21 **styled theming** (`@primeuix/themes` Aura preset + `cssLayer`) + Tailwind v4 + styles.css design tokens
- **Authentication:** In-memory JWT (signal in `TokenStorageService`) + functional interceptor with refresh (NEVER localStorage)
- **Validation:** Angular Reactive Forms with Signals
- **Forms:** Angular Reactive Forms
- **Tables:** PrimeNG Table (styled theming + Pass Through)
- **UI Components:** PrimeNG v21 components (styled theming)
- **Icons:** PrimeIcons or custom SVG icons
- **Toasts:** PrimeNG Message/Toast service
- **HTTP:** HttpClient (Fetch API is default in v22 — no `withFetch()` needed) with functional interceptors
- **API client:** Generated by `ng-openapi-gen` into `src/app/api`
- **Validation/Sanitization:** Zod `^4` (runtime validation) + DOMPurify `^3` (HTML sanitization) — both installed
- **Testing:** Vitest (`vitest` + `jsdom`) — Playwright is NOT used

## Commands
- `npm start` / `ng serve` - Start development server
- `npm run build` / `ng build` - Production build

- `npx tsc --noEmit` - Type-check (strict)
- `npm run generate:api` - Regenerate the API client from the backend OpenAPI spec

## Important Context

### Component Rules
- All components are standalone by default (NO `standalone: true` needed in v22)
- **OnPush is the default in v22 — do NOT set it explicitly in new components** (redundant). Existing explicit OnPush is safe to keep.
- **`ChangeDetectionStrategy.Default` is `@deprecated`** in v22 (now an alias for `ChangeDetectionStrategy.Eager`, due to be removed) — use `Eager` and migrate any code referencing `Default`
- Use `@Service()` instead of `@Injectable({ providedIn: 'root' })` for new singleton services
- Use `injectAsync()` for lazy, code-split service loading
- Use `input()` and `output()` functions instead of decorators
- Use signals for state management (signal, computed, effect)
- Use `computed()` for derived state
- Use native control flow (`@if`, `@for`, `@switch`) instead of `*ngIf`, `*ngFor`, `*ngSwitch`
- v22 templates support **spread** (`{{ fn(...sig(), x) }}`), **arrow functions** in event bindings, **exhaustive `@switch`** (`@default { never; }`), and comments inside elements — keep complex logic in the class
- Use `@defer` for lazy loading heavy components
- Prefer Reactive Forms or **Signal Forms** (stable in v22) over Template-driven
- Use `inject()` instead of constructor injection
- Put host bindings in `host` object of decorator, NOT `@HostBinding`/`@HostListener`
- Use `NgOptimizedImage` for static images (not base64)

### Data Fetching
- Use `resource()` for Promise-returning async, `rxResource()` for Observable-returning async
- Use `httpResource()` for declarative HTTP signal fetching — **stable in v22** (import from `@angular/common/http`)
- Use `debounced()` to debounce signal values (stable in v22) — e.g. for search/filter inputs
- Use `injectAsync()` for lazy, code-split service injection (load only when called)
- Router `paramsInheritanceStrategy: 'always'` is the v22 default — bind route params as `input.required<string>()`
- Services use `providedIn: 'root'` for singletons
- Use functional interceptors for auth tokens
- Generated OpenAPI types are compile-time only; validate trust-boundary data at runtime with Zod (`zod ^4`, installed) — see OWASP skill
- Use `resource().isLoading()` for loading state and `resource().error()` for errors
- NEVER use `any` - use `unknown` when type is uncertain

### Styling Rules
- Maintain existing styles.css design tokens (CSS custom properties)
- Use PrimeNG v21 styled theming (`@primeuix/themes`); customize tokens with `definePreset`
- Keep `cssLayer` enabled so Tailwind v4 utilities override PrimeNG cleanly
- Attach styles.css classes to PrimeNG components via the `pt` Pass Through API
- NEVER use hex values, ALWAYS use CSS custom properties (`var(--token)`)
- Before implementing any component, read `styles.css` and use existing tokens
- If a token doesn't exist, add it to `styles.css` first
- Dark-first: `:root`/`.dark` = dark, `[data-theme="light"]` = light; `darkModeSelector: '.dark'`
- Use class bindings instead of `ngClass`
- Use style bindings instead of `ngStyle`

### PrimeNG v21 Theming Configuration
```typescript
// src/app/app.config.ts
import { providePrimeNG } from 'primeng/config';
import Aura from '@primeuix/themes/aura';   // NOT 'primeng/themes'

providePrimeNG({
  theme: {                                   // styled mode — no `unstyled: true`
    preset: Aura,
    options: {
      darkModeSelector: '.dark',
      cssLayer: { name: 'primeng', order: 'primeng' }
    }
  }
})
```

### Security
- NEVER store JWT in localStorage/sessionStorage — this app keeps the access token in-memory (signal)
- Use functional interceptors (`HttpInterceptorFn`) for auth — never class-based interceptors
- Set CSP + security headers at the **server/SSR** layer (`server.ts`), NEVER via an HTTP request interceptor
- Sanitize user-controlled HTML with DOMPurify before `[innerHTML]`
- Follow the OWASP skill (frontend security baseline)
- Never expose stack traces to clients; no `console.log` in production
- Use environment variables for config; never hardcode external URLs

### Accessibility (WCAG 2.2 AA)
- Focus rings: `outline: 2px solid var(--accent-primary); outline-offset: 2px`
- Text contrast ≥ 4.5:1 (normal), ≥ 3:1 (large)
- Icon-only buttons MUST have `aria-label` or `title`
- Modals close on `Escape`
- Minimum 24×24px tap targets
- MUST pass all AXE checks

## Don'ts
- NEVER use NgModules - all components must be standalone
- NEVER use `@HostBinding`/`@HostListener` - use host object in decorator
- NEVER use `ngClass`/`ngStyle` - use class/style bindings
- NEVER use `*ngIf`/`*ngFor`/`*ngSwitch` - use `@if`/`@for`/`@switch`
- NEVER use `any`, `@ts-ignore`, or `as unknown as X` - use `unknown`
- NEVER store JWT in localStorage
- NEVER use `console.log` in production code
- NEVER use pipes for complex transformation - use `computed()`
- NEVER use async pipe excessively - prefer signal binding
- NEVER use `mutate` on signals - use `update` or `set`
- NEVER set `changeDetection: ChangeDetectionStrategy.OnPush` in new components (v22 default)
- NEVER use `ChangeDetectionStrategy.Default` — `@deprecated` in v22 (alias for `Eager`, due to be removed); use `ChangeDetectionStrategy.Eager`
- NEVER use `withFetch()` — Fetch is now the default HTTP transport in v22
- NEVER use `@Injectable({ providedIn: 'root' })` for new services — use `@Service()` instead
